# Rol Beklentileri

Bu bölümde Backend Engineer, Senior Backend Engineer, Software Architect ve Team Lead rollerinin mülakattaki beklentileri karşılaştırılır.

## Backend Engineer

### Beklenen Teknik Seviye
- 2-5 yıl deneyim
- .NET ve ASP.NET Core ekosisteminde rahat
- Temel database knowledge (PostgreSQL)
- Redis gibi cache tools kullanımı
- Basic distributed systems concepts
- API geliştirme ve RESTful design

### Mülakatta Ölçülen Konular
- .NET fundamentals (CLR, GC, async/await)
- ASP.NET Core middleware pipeline
- Database querying ve indexing
- Redis caching basics
- Basic Kafka/message queue usage
- API design ve REST principles
- Error handling ve logging

### Güçlü Anlatmam Gerekenler
- .NET ve ASP.NET Core projelerim
- API geliştirme deneyimim
- Database optimization çalışmam
- Redis cache implementasyonum
- Basic distributed systems deneyimim
- Problem çözme yeteneğim

### Kaçınmam Gereken Cevaplar
- "Sadece CRUD API yazdım."
- "Database tuning yapmadım."
- "Redis kullanmadım."
- "Distributed systems bilmiyorum."
- "Sadece frontend developer gibi çalıştım."

### Hazırlık Öncelikleri
1. .NET ve ASP.NET Core fundamentals
2. PostgreSQL basics ve indexing
3. Redis caching strategies
4. API design principles
5. Basic Kafka concepts

## Senior Backend Engineer

### Beklenen Teknik Seviye
- 5-8 yıl deneyim
- Deep .NET knowledge
- Advanced database optimization
- Strong distributed systems understanding
- System design capability
- Performance optimization
- Fault tolerance ve resilience

### Mülakatta Ölçülen Konular
- Advanced .NET (memory management, threading)
- Distributed systems (CAP, consistency, fault tolerance)
- Kafka advanced (offset management, idempotency, DLQ)
- Redis advanced (data structures, clustering)
- PostgreSQL advanced (indexing, query optimization)
- System design (scalability, capacity planning)
- Performance optimization (profiling, bottleneck analysis)
- Observability (logging, metrics, tracing)

### Güçlü Anlatmam Gerekenler
- Complex distributed systems deneyimim
- Performance optimization başarılarım
- System design decisions
- Fault tolerance implementasyonlarım
- Team collaboration ve mentorship
- Technical leadership deneyimim

### Kaçınmam Gereken Cevaplar
- "Sadece basic API'lar geliştirdim."
- "System design yapmadım."
- "Performance tuning bilmiyorum."
- "Distributed systems theory bilmem."
- "Junior seviyede çalıştım."

### Hazırlık Öncelikleri
1. Distributed systems (CAP, consistency, fault tolerance)
2. Kafka advanced (offset, idempotency, DLQ)
3. System design (scalability, capacity planning)
4. Performance optimization
5. Observability (logging, metrics, tracing)

- Technology selection expertise

### Mülakatta Ölçülen Konular
- Architecture patterns (microservices, event-driven, CQRS)

- Technology selection ve trade-off analysis
- Integration patterns (API gateway, service mesh)

- Technology selection süreçlerim
