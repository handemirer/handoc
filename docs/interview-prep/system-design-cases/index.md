# System Design Case'leri

Bu bölümde mülakatlarda sorulabilecek system design case'leri yer alır. Her case için problem, gereksinimler, high-level design, data model, API tasarımı, scalability, reliability ve trade-off'lar detaylandırılır.

## Case'ler

- [Real-Time Market Data System](realtime-market-data-system.md) - BIST ve global piyasalar için gerçek zamanlı market data sistemi
- [Notification System](notification-system.md) - Büyük ölçekli bildirim sistemi
- [Log Processing System](log-processing-system.md) - Log toplama ve işleme sistemi
- [URL Shortener](url-shortener.md) - URL kısaltma servisi
- [Stock Price API](stock-price-api.md) - Hisse senedi fiyat API'si

## Her Case'in İçeriği

Her system design case sayfasında şu bölümler yer alır:

1. **Problem** - Çözülen problem
2. **Gereksinimler** - Functional ve non-functional requirements
3. **Varsayımlar** - Sistem varsayımları
4. **High-Level Design** - Yüksek seviye mimari
5. **Data Model** - Veri modeli
6. **API Tasarımı** - API endpoint'leri
7. **Cache Stratejisi** - Caching approach
8. **Message Queue / Event Flow** - Event-driven architecture
9. **Scalability** - Ölçeklenebilirlik
10. **Reliability** - Güvenilirlik
11. **Observability** - Monitoring ve tracing
12. **Trade-Offs** - Tasarım trade-off'ları
13. **Mülakatta Nasıl Anlatırım?** - Anlatım stratejisi
14. **Kırmızı Bayrak Cevaplar** - Kaçınılması gereken cevaplar

## Çalışma Stratejisi

1. Her case'i 30-45 dakikada çöz
2. High-level design'dan başla
3. Data model ve API tasarımı detaylandır
4. Scalability ve reliability düşün
5. Trade-off'ları açıkla
6. Mülakat anlatım stratejisi hazırla
