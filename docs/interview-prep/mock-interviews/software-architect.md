# Software Architect Mock Interview

## Amaç

Software Architect rolü için architectural knowledge, technology selection, system design at scale ve strategic technical decisions yetkinliklerini ölçmek.

## Beklenen Seviye

- 8+ yıl deneyim
- Deep architectural knowledge
- Cross-system integration
- Technology selection expertise
- Organizational impact

## Isınma Soruları

1. Architect rolü nedir?
2. Büyük system tasarım deneyimin?
3. Technology selection sürecin?
4. Cross-team çalışma deneyimin?

## Güçlü Cevap Örnekleri

**Microservices vs Monolith:**
"Microservices independent scaling ve deployment sağlar ancak distributed complexity ekler. Monolith simpler ancak limited scaling. Start with monolith, evolve to microservices based on team size and domain complexity."

**Technology Selection:**
"Team expertise, ecosystem maturity, performance, scalability, cost. .NET için strong ecosystem, good performance, enterprise support. Go için high performance, simple deployment. Java için large ecosystem, enterprise adoption."

## Zayıf Cevap Örnekleri

- "Her zaman microservices kullanırım."
- "Technology selection bilmiyorum, team lead verir."
- "Architecture pattern bilmiyorum."
- "Cross-team çalışmadım, sadece single service."
