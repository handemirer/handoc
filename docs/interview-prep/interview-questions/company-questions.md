# Şirkete Sorulacak Sorular

Bu bölümde adayın şirkete sorabileceği sorular kategorilere ayrılmıştır.

## Teknik Kültür

- Teknik stack nedir? (.NET, Java, Go, vs.)
- Technology selection nasıl yapılır?
- Yeni teknoloji adoption süreci nasıl?
- Technical debt nasıl yönetiliyor?
- Innovation için fırsat var mı?
- R&D time var mı?

## Code Review

- Code review process nasıl?
- Code review frequency nedir?
- PR approval process nedir?
- Automated testing requirements neler?
- Code quality standards neler?
- Linting ve static analysis kullanılıyor mu?

## Mentorluk

- Mentorship program var mı?
- Junior developer'lara support nasıl?
- Learning budget var mı?
- Conference attendance destekleniyor mu?
- Internal tech talk'ler var mı?
- Knowledge sharing culture nasıl?

## Deployment Süreci

- CI/CD pipeline nasıl?
- Deployment frequency nedir?
- Canary deployment veya blue-green deployment kullanılıyor mu?
- Rollback process nasıl?
- Feature flags kullanılıyor mu?
- Environment'lar (dev, staging, prod) nasıl yönetiliyor?

## Monitoring / Observability

- Monitoring tools neler? (Prometheus, Datadog, vs.)
- Logging infrastructure nasıl?
- Distributed tracing kullanılıyor mu?
- Alerting process nasıl?
- SLO/SLA tanımlı mı?
- On-call process nasıl?

## Test Kültürü

- Test coverage hedefi var mı?
- Unit test, integration test, E2E test oranı nasıl?
- Test automation seviyesi nedir?
- Load testing yapılıyor mu?
- QA team var mı?
- Test environment nasıl?

## Ürün-Yazılım İlişkisi

- Product ve engineering collaboration nasıl?
- Product roadmap nasıl belirleniyor?
- Engineer'lar product decision'lara katılıyor mu?
- User feedback nasıl toplanıyor?
- Data-driven decision making nasıl?
- Technical vs product priority nasıl dengeleniyor?

## Kariyer Gelişimi

- Career path nasıl? (Individual contributor vs management)
- Promotion process nasıl?
- Performance review frequency nedir?
- Internal mobility var mı?
- Skill development destekleniyor mu?
- Leadership opportunities var mı?

## İlk 3 Ay Beklentisi

- İlk 3 ayda ne başarmak bekleniyor?
- Onboarding process nasıl?
- Buddy system var mı?
- Documentation seviyesi nasıl?
- İlk project ne olacak?
- Success criteria nasıl tanımlanmış?

## Teknik Borç Yönetimi

- Technical debt nasıl takip ediliyor?
- Refactoring time ayrılıyor mu?
- Technical debt prioritization nasıl?
- Legacy code migration nasıl yönetiliyor?
- Tech stack upgrade planları var mı?
- Decommission planları nasıl?

## Ek Sorular

- Remote/hybrid work policy nasıl?
- Work-life balance culture nasıl?
- Team size ve structure nasıl?
- Cross-team collaboration nasıl?
- Company culture nasıl tanımlanır?
- Son 1 yılda ne değişti?
- Gelecek 1 yılda hedefler neler?

## Soru Sorma İpuçları

- **Doğal sor**: Ezber gibi değil, doğal akışta sor
- **Context-based**: Önceki konuşmaya dayalı sor
- **Show interest**: Şirketi gerçekten merak ettiğini göster
- **Avoid negative**: Negatif veya aggressive sorulardan kaçın
- **Timing**: Doğru zamanı seç (genelde sonunda)

## Örnek Soru Akışı

1. **Early**: Teknik stack, technology selection
2. **Middle**: Code review, deployment, testing
3. **Late**: Career growth, culture, expectations
